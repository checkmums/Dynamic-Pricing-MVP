
# Dynamic Pricing MVP

## Описание проекта
Алгоритм динамического ценообразования для розничной торговли. MVP определяет оптимальную цену товара на основе исторических данных о продажах и ценах конкурентов.

## Установка и запуск

### 
```bash
1. Создание виртуального окружения
python3 -m venv venv
source venv/bin/activate

2. Установка зависимостей
bash
pip install -r requirements.txt

3. Генерация данных
bash
python3 generate_data.py

4. Запуск веб-интерфейса
bash
streamlit run app.py
После запуска откроется браузер с интерфейсом. Нажмите кнопку "Сгенерировать новые данные" и перейдите во вкладку "Рекомендации".

Алгоритм работы

Генератор данных

Создаёт синтетические данные со следующими параметрами:

3 товара
100 дней истории
Зависимость продаж от цены (эластичность)
Влияние цены конкурента на спрос
Формула генерации:

text
sales = 150 - 0.5 * our_price + 0.3 * (competitor_price - our_price) + шум
Умный алгоритм ценообразования

Обучает линейную регрессию на исторических данных:

text
sales = a + b * price + c * competitor_price
Находит оптимальную цену через максимизацию выручки:

text
revenue = price * (a + b * price + c * competitor_price)
price_opt = -(a + c * competitor_price) / (2 * b)
Применяет ограничения (цена не ниже 70% и не выше 130% от исторического диапазона)
Простой алгоритм (для сравнения)

Использует эвристические правила:

Если конкурент дешевле на 10% → снижаем цену на 5%
Если продажи низкие → снижаем цену на 3%
Результаты

Алгоритм	Рост выручки
Простой (эвристика)	-2.8%
Умный (машинное обучение)	+47.1%
На тестовых данных умный алгоритм увеличил выручку на 47.1%

Выводы

В ходе работы над проектом:

Разработан генератор реалистичных данных с контролируемой эластичностью
Реализованы два подхода к ценообразованию: эвристический и на основе ML

Создан удобный веб-интерфейс для демонстрации работы алгоритма
Ограничения и пути развития

Текущие ограничения:

Используется линейная модель спроса (в реальности зависимость может быть нелинейной)
Не учитываются акции, сезонность и остатки товара
Конкуренты не реагируют на изменение наших цен
Направления улучшения:

Добавление нелинейных моделей (логарифмическая эластичность)
Учёт сезонных коэффициентов
Симуляция реакции конкурентов
Оптимизация не по выручке, а по маржинальной прибыли
